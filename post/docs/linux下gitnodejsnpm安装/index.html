linux下git&nodeJS&npm安装&hugo部署

1. git 安装

1.安装前检查

#1.查看git版本,如果能出版本号，说明git已经安装，酌情考虑是否需要重新安装
git --version
#2.卸载git
yum remove git

下载 git ，然后解压，编译安装，配置。

#在local下建立git文件夹 查看当前目录 我的控制台打印出 /usr/local/git 
pwd

# shell下载
wget https://github.com/git/git/archive/v2.23.0.tar.gz

# 解压
tar -xzvf git-2.18.1.tar.gz

#进入解压目录
cd git-2.18.1

#配置git安装路径
./configure prefix=/usr/local/git/git-2.18.1

#编译安装
make && make install

#注意 如果安装失败，则执行以下指令安装依赖包
yum install curl-devel expat-devel gettext-devel openssl-devel zlib-devel 
yum install gcc-c++ perl-ExtUtils-MakeMaker

#最后配置环境变量，将git的运行指令添加到环境变量中
    # 在 /etc/profile 最后一行加入
    export PATH=$PATH:/usr/local/git/bin
 #或
 	echo "export PATH=$PATH:/usr/git/bin" >> /etc/profile

#使配置生效
source /etc/profile

# 查看git版本 
git --version

2. nodejs&npm

根据需要下载 nodeJs ，解压。以我的运行环境为例。

[root@localhost node]# ls
node-v12.9.1-linux-x64  node-v12.9.1-linux-x64.tar.xz
[root@localhost node]# cd node-v12.9.1-linux-x64/
[root@localhost node-v12.9.1-linux-x64]# ls
bin  CHANGELOG.md  include  lib  LICENSE  README.md  share
[root@localhost node-v12.9.1-linux-x64]# cd bin
[root@localhost bin]# ls
node  npm  npx

配置软连接

[root@localhost bin]# ln -s /usr/local/node/node-v12.9.1-linux-x64/bin/npm   /usr/local/bin/
[root@localhost bin]# ln -s /usr/local/node/node-v12.9.1-linux-x64/bin/npm   /usr/local/bin/

安装成功，查看版本

[root@localhost bin]# node -v
v12.9.1
[root@localhost bin]# npm -v
6.10.2

3. hugo安装及github部署

到 Hugo Releases 下载 *Linux-64bit.tar.gz 压缩包放置到 /usr/local/hugo下解压，生成一个名为 hugo的可执行文件。主题本人使用的是 hugo-theme-jane ，hugo网站创建的具体步骤可参考github上jane的教程说明。在这里主要讲的是github部署。

首先在github上创建一个名为 xxx.github.io的仓库，其中 xxx 为你的github用户名。然后将你hugo目录下的public文件夹通过git上传到github上即可。可参考如下过程:

$ mkdir /usr/blog/git
$ git clone '刚你创建的空仓库'
$ cd /usr/local/hugo/myblog/public
$ cp -r . /usr/blog/git
$ cd /usr/blog/git/
$ git add .
$ git commit -m "初始化hugo"
$ git push origin master

在启动hugo生成站点时，需要注意 一般的启动方式hugo server并不适用于远程部署。

如果要部署到github，可参考如下编译方式：

# jane 是主题名，记住 .io/ 中的 / 不能缺少
hugo --theme=jane --baseUrl=https://xxx.github.io/

# 如果hugo指令不能用，可以配置环境变量或者软连接
[root@localhost bin]# ln -s /usr/local/hugo/hugo /usr/local/bin/
