idea快捷键使用

1. 快捷键

  1. ctrl + n == 搜索类、方法、属性
  2. ctrl + shift + n == 搜索文件
  3. **ctrl + alt +n == 代码提示 **
  4. / ctrl + alt + 空格 = 代码提示
  5. alt + f7 == 查询在某处使用
  6. ctrl + q == 查看代码提示及内容
  7. ctrl + b == 查看类的声明 ==ctrl+左键
  8. ctrl + f12 == 查看文件列表（类的方法列表）
  9. ctrl + alt + ⬆/⬇ = 查看 父类/子类
  10. ctrl + shift + ⬆/⬇ = 上移/下移一行代码
  11. 连续两次shift == 查找文件
  12. 连安两下 shift 查找文件
  13. CtrL +shift +r 全局搜索
  14. ctrl +r 搜索替换当前页的单词
  15. 查看类继承关系图：Ctrl+Alt+U
  16. 查看定义的变量在哪里被调用：Ctrl+Alt+F7
  17. 查看Java方法调用树(被调/主调)：Ctrl+Alt+H
  18. **在codeStyle_java_import设置中，可以设置import多少个类自动变成import ***
  19. ctrl +alt +v 自动生成返回值
  20. ctrl + alt +t 生成环绕代码块 surround with
  21. ctl + alt + <- 或 -> 上一步或下一步
  22. ctrl + alt + m 抽取代码为方法
  23. ctrl + e 打开最近文件
  24. ctrl + h 查看类的继承关系
  25. ctr + shift + u 大小写转化
  26. shift + enter 向下插入空行
  27. ctrl + alt + enter 向上插入空行

2.使用技巧

debug篇

  1. 右键断点可以设置进入断点条件
  2. ctrl + u 进入查看debug时表达式的值

RabbitMQ学习笔记

1. 基本概念

  1. 如果使用默认方式接收队列，先启动多个监听者，再启动生产者，消费方式默认为轮询分发模式，不论监听者执行效率多少，都是轮询获取mq消息。

  2. 使用多劳模式时，必须手动确认ack。

    // 能者多劳关键代码
//1. 在队列声明之后
channel.basicQos(1);
//2. 在消费之后
finally {
         channel.basicAck(envelope.getDeliveryTag(), false);
}
//3.定义消费者
channel.basicConsume(queueName, false, consumer);

  3. 订阅模式、

多个队列绑定到同一个交换机上，生产者将消息发送到交换机，由消费者订阅消息。

  1. 路由模式

在声明exchange时，给予路由关键字，在在channel绑定exchange时，也声明路由，在生产者将数据投递到exchange上后，exchange根据路由关键字将数据投递给channel。

注意exchange 类型 type值为 direct

关键代码：

//生产者
 Connection connection = ConnectionUtils.getConnection();
        Channel channel = connection.createChannel();
        channel.exchangeDeclare(exchange, "direct");
        channel.basicQos(1);
        String routingKey = "error";
        for (int i = 0; i < 50; i++) {
            String msg = sendStr + i;
            channel.basicPublish(exchange, routingKey, null, msg.getBytes());
        }
        channel.close();
        connection.close();

  1. topic模式

根据routingKey上的通配符分发事件。*代表匹配一个关键字，#代表匹配多个关键字.

关键代码：

//服务端
 channel.exchangeDeclare(exchange, "topic");
        channel.basicQos(1);
        String routingKey = "goods，type.add";
        for (int i = 0; i < 50; i++) {
            String msg = sendStr + i;
            channel.basicPublish(exchange, routingKey, null, msg.getBytes());
        }

//客户端1,queueName队列可以收到消息
channel.exchangeDeclare(exchange, "topic");
channel.queueDeclare(queueName, false, false, false, null);
channel.queueBind(queueName, exchange, "goods.*.add");

//客户端2,queueName队列可以收到消息
channel.exchangeDeclare(exchange, "topic");
channel.queueDeclare(queueName, false, false, false, null);
channel.queueBind(queueName, exchange, "goods.#");

//客户端3,queueName队列不可以收到消息
channel.exchangeDeclare(exchange, "topic");
channel.queueDeclare(queueName, false, false, false, null);
channel.queueBind(queueName, exchange, "goods.type.delete");
  1. 事务确认机制

通过事务控制操作mq队列，关键代码如下。

try{
                channel.txSelect();
                channel.basicPublish("", queueName, null, msg.getBytes());
                //如果抛出异常，事件不发送到mq
                //int a = 1/0;
                channel.txCommit();
            }catch (Exception e){
                System.out.println("channel rollback...");
                channel.txRollback();
            }

  1. conform模式确认机制

    //方式1 单条 confirm
//生产者开启confirm模式
channel.confirmSelect();
for (int i = 0; i < 50; i++) {
String msg = sendStr + i;
channel.basicPublish("", queueName, null, msg.getBytes());
try{
    if(channel.waitForConfirms()){

    } else{
        System.out.println("消息发送失败");
    }
} catch (Exception e){
    e.printStackTrace();
}
}
